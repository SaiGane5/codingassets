MongoDB

Let’s delve into MongoDB and Mongoose through a project-based approach. We will build a Contact Management Application to demonstrate the essential features of MongoDB and how to use Mongoose effectively.

MongoDB Tutorial

1. CRUD Operations

CRUD stands for Create, Read, Update, and Delete, which are the four basic operations for managing data.

Create

You can insert new documents into a collection using insertOne() or insertMany().

Example:

const MongoClient = require('mongodb').MongoClient;
const url = 'mongodb://localhost:27017';
const client = new MongoClient(url);

async function run() {
    await client.connect();
    const database = client.db('contactDB');
    const contacts = database.collection('contacts');

    const newContact = { name: 'John Doe', email: 'john@example.com' };
    const result = await contacts.insertOne(newContact);
    console.log(`New contact created with the following id: ${result.insertedId}`);
}

run().catch(console.dir);

Read

You can retrieve documents using find() and findOne().

Example:

const contact = await contacts.findOne({ name: 'John Doe' });
console.log(contact);

Update

You can modify existing documents using updateOne() or updateMany().

Example:

const updateResult = await contacts.updateOne(
    { name: 'John Doe' },
    { $set: { email: 'john.doe@example.com' } }
);
console.log(`${updateResult.modifiedCount} document(s) updated`);

Delete

You can remove documents using deleteOne() or deleteMany().

Example:

const deleteResult = await contacts.deleteOne({ name: 'John Doe' });
console.log(`${deleteResult.deletedCount} document(s) deleted`);

2. Aggregation

Aggregation allows you to process data records and return computed results. The aggregation framework provides various stages, such as $match, $group, $sort, etc.

Example:

const aggregationResult = await contacts.aggregate([
    { $group: { _id: '$email', count: { $sum: 1 } } }
]).toArray();
console.log(aggregationResult);

3. Indexing

Indexing improves the performance of database queries. By creating indexes on certain fields, MongoDB can quickly locate and access documents.

Example:

await contacts.createIndex({ email: 1 }); // Creates an index on the 'email' field

4. Atlas Setup

MongoDB Atlas is a cloud-hosted database service. You can create a free-tier database on Atlas for testing and development.

Steps:

  1. Sign up for a MongoDB Atlas account.
  1. Create a new cluster.
  1. Whitelisting your IP: Allow your IP address in the network access settings.
  1. Create a database user: Set a username and password.
  1. Connect to the cluster using the provided connection string in your Node.js application.

Example:

const url = 'mongodb+srv://<username>:<password>@cluster.mongodb.net/contactDB?retryWrites=true&w=majority';
const client = new MongoClient(url);

Mongoose Tutorial

1. Schemas

Schemas define the structure of documents within a collection.

Example:

const mongoose = require('mongoose');

const contactSchema = new mongoose.Schema({
    name: { type: String, required: true },
    email: { type: String, required: true, unique: true }
});

const Contact = mongoose.model('Contact', contactSchema);

2. Models

Models are constructors compiled from Schemas and allow you to create and query documents.

Example:

const newContact = new Contact({ name: 'Jane Doe', email: 'jane@example.com' });
await newContact.save(); // Saves the contact to the database

3. Validation

Mongoose provides built-in validation for schema fields.

Example:

const invalidContact = new Contact({ name: 'Invalid' });
try {
    await invalidContact.save();
} catch (error) {
    console.error('Validation Error:', error.message);
}

4. Middleware

Middleware are functions that can be executed at specific points during the lifecycle of a document, like save, validate, remove, etc.

Example:

contactSchema.pre('save', function (next) {
    console.log('A contact is about to be saved:', this);
    next(); // Proceed to the next middleware or save
});

Project: Contact Management Application

Step 1: Setup the Project

  1. Create a new directory and initialize your project:
    mkdir contact-management-app
cd contact-management-app
npm init -y
npm install mongoose express

Step 2: Create the Application

app.js: Main application logic

const express = require('express');
const mongoose = require('mongoose');
const app = express();

// Connect to MongoDB
mongoose.connect('mongodb://localhost:27017/contactDB', { useNewUrlParser: true, useUnifiedTopology: true });

// Define Schema and Model
const contactSchema = new mongoose.Schema({
    name: { type: String, required: true },
    email: { type: String, required: true, unique: true }
});

contactSchema.pre('save', function (next) {
    console.log('A contact is about to be saved:', this);
    next();
});

const Contact = mongoose.model('Contact', contactSchema);

// Middleware
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// CRUD Operations
app.post('/contacts', async (req, res) => {
    const newContact = new Contact(req.body);
    try {
        await newContact.save();
        res.status(201).send('Contact created');
    } catch (error) {
        res.status(400).send('Error: ' + error.message);
    }
});

app.get('/contacts', async (req, res) => {
    const contacts = await Contact.find();
    res.json(contacts);
});

// Update and Delete
app.put('/contacts/:id', async (req, res) => {
    const contact = await Contact.findByIdAndUpdate(req.params.id, req.body);
    res.send('Contact updated');
});

app.delete('/contacts/:id', async (req, res) => {
    await Contact.findByIdAndDelete(req.params.id);
    res.send('Contact deleted');
});

// Aggregation Example
app.get('/contacts/aggregation', async (req, res) => {
    const aggregationResult = await Contact.aggregate([
        { $group: { _id: '$email', count: { $sum: 1 } } }
    ]);
    res.json(aggregationResult);
});

// Start server
app.listen(3000, () => {
    console.log('Server running on <http://localhost:3000>');
});

Step 3: Run the Application

  1. Run your application:
    node app.js

Step 4: Test the Application

You can use tools like Postman or curl to test your CRUD operations.
