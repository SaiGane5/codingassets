React Notes with foundation project

Let's explore the fundamentals and advanced concepts of React step by step, using a project to integrate all these topics. Our project will be a Simple Todo List Application where users can add, delete, and filter tasks.

Fundamentals of React

1. Components & Props

Components are the building blocks of a React application. They can be functional or class-based. Props (short for properties) allow you to pass data to components.

Example:

// Functional Component
const Greeting = ({ name }) => {
    return <h1>Hello, {name}!</h1>;
};

// Using the Greeting component
<Greeting name="Alice" />

Usage in Todo List:
We’ll create a
Todo component to display individual tasks and pass task data as props.

2. State & Lifecycle

State allows components to manage dynamic data that can change over time. Lifecycle methods enable you to run code at specific points in a component’s life.

Example:

class Timer extends React.Component {
    constructor(props) {
        super(props);
        this.state = { seconds: 0 };
    }

    componentDidMount() {
        this.interval = setInterval(() => {
            this.setState({ seconds: this.state.seconds + 1 });
        }, 1000);
    }

    componentWillUnmount() {
        clearInterval(this.interval);
    }

    render() {
        return <p>Seconds: {this.state.seconds}</p>;
    }
}

Usage in Todo List:
We’ll manage the list of tasks in the state and update it as users add or remove tasks.

3. Event Handling

Event handling in React is done through camelCase syntax. You pass a function to the event handler.

Example:

class Button extends React.Component {
    handleClick = () => {
        alert("Button clicked!");
    };

    render() {
        return <button onClick={this.handleClick}>Click me</button>;
    }
}

Usage in Todo List:
We’ll handle events for adding and deleting tasks, updating the state accordingly.

4. Conditional Rendering

Conditional rendering allows you to render different components or elements based on the state.

Example:

const LoginLogout = ({ isLoggedIn }) => {
    return isLoggedIn ? <button>Logout</button> : <button>Login</button>;
};

Usage in Todo List:
We’ll conditionally render a message when there are no tasks in the list.

Advanced React Concepts

1. Hooks (useState, useEffect)

Hooks are functions that let you use state and other React features in functional components.

  • useState: Allows you to add state to a functional component.

    Example:

    const [count, setCount] = useState(0);
  • useEffect: Enables you to perform side effects in functional components.

    Example:

    useEffect(() => {
    document.title = `Count: ${count}`;
}, [count]);

Usage in Todo List:
We’ll use
useState for managing tasks and useEffect for updating the document title based on the number of tasks.

2. Context API

The Context API provides a way to share values like state between components without passing props manually at every level.

Example:

const MyContext = React.createContext();

const MyProvider = ({ children }) => {
    const [value, setValue] = useState("Hello");
    return (
        <MyContext.Provider value={{ value, setValue }}>
            {children}
        </MyContext.Provider>
    );
};

// Consuming the context
const MyComponent = () => {
    const { value } = useContext(MyContext);
    return <div>{value}</div>;
};

Usage in Todo List:
We’ll use the Context API to manage the theme of the application.

3. Custom Hooks

Custom hooks allow you to encapsulate logic that can be reused across components.

Example:

const useFetch = (url) => {
    const [data, setData] = useState(null);

    useEffect(() => {
        const fetchData = async () => {
            const response = await fetch(url);
            const result = await response.json();
            setData(result);
        };
        fetchData();
    }, [url]);

    return data;
};

Usage in Todo List:
We can create a custom hook for managing tasks, encapsulating the logic for adding and deleting tasks.

4. Performance Optimization

Performance optimization in React can be achieved through techniques like memoization with React.memo or useMemo and avoiding unnecessary re-renders.

Example:

const MemoizedComponent = React.memo(({ value }) => {
    return <div>{value}</div>;
});

Usage in Todo List:
We can optimize the rendering of the task list by memoizing the
Todo component.

3. Create Components and Context

Create the following files in the src directory:

src/App.js

import React, { useEffect } from 'react';
import TodoList from './TodoList';
import { ThemeProvider } from './ThemeContext';
import useTasks from './useTasks';
import './styles.css'; // Optional for styling

const App = () => {
    const { tasks, addTask, deleteTask } = useTasks();

    useEffect(() => {
        const savedTasks = JSON.parse(localStorage.getItem('tasks'));
        if (savedTasks) {
            tasks.setTasks(savedTasks);
        }
    }, []);

    useEffect(() => {
        localStorage.setItem('tasks', JSON.stringify(tasks));
    }, [tasks]);

    const handleKeyDown = (event) => {
        if (event.key === 'Enter' && event.target.value) {
            addTask(event.target.value);
            event.target.value = ''; // Clear the input field
        }
    };

    return (
        <ThemeProvider>
            <h1>Todo List</h1>
            <input type="text" onKeyDown={handleKeyDown} placeholder="Add a new task" />
            <TodoList tasks={tasks} deleteTask={deleteTask} />
        </ThemeProvider>
    );
};

export default App;

src/TodoList.js

import React from 'react';
import Todo from './Todo';

const TodoList = ({ tasks, deleteTask }) => {
    return (
        <ul>
            {tasks.length === 0 ? (
                <p>No tasks available. Add some!</p>
            ) : (
                tasks.map((task) => (
                    <Todo key={task.id} task={task.task} deleteTask={() => deleteTask(task.id)} />
                ))
            )}
        </ul>
    );
};

export default TodoList;

src/Todo.js

import React from 'react';

const Todo = React.memo(({ task, deleteTask }) => {
    return (
        <li>
            {task}
            <button onClick={deleteTask}>Delete</button>
        </li>
    );
});

export default Todo;

src/ThemeContext.js

import React, { createContext, useContext, useState } from 'react';

const ThemeContext = createContext();

export const ThemeProvider = ({ children }) => {
    const [theme, setTheme] = useState('light');

    const toggleTheme = () => {
        setTheme((prevTheme) => (prevTheme === 'light' ? 'dark' : 'light'));
    };

    return (
        <ThemeContext.Provider value={{ theme, toggleTheme }}>
            <div className={theme}>{children}</div>
        </ThemeContext.Provider>
    );
};

export const useTheme = () => useContext(ThemeContext);

src/useTasks.js

import { useState } from 'react';

const useTasks = () => {
    const [tasks, setTasks] = useState([]);

    const addTask = (task) => {
        setTasks((prevTasks) => [...prevTasks, { id: Date.now(), task }]);
    };

    const deleteTask = (id) => {
        setTasks((prevTasks) => prevTasks.filter((task) => task.id !== id));
    };

    return { tasks, addTask, deleteTask, setTasks }; // Return setTasks to use it in App.js
};

export default useTasks;
