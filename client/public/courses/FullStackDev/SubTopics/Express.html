Express Notes

Let's explore foundational concepts of Express.js through a simple Task Management Application. This application will demonstrate the key features of Express, including routing, middleware, error handling, template engines, static files, and database integration.

1. Routing

Routing in Express allows you to define various endpoints that the application can respond to. Each route can have one or more handler functions.

How It Works:

  • Use app.get(), app.post(), etc., to define routes.
  • Each route can have a callback function that handles the request and response.

Example:

const express = require('express');
const app = express();

app.get('/tasks', (req, res) => {
    res.send('List of tasks');
});

app.post('/tasks', (req, res) => {
    res.send('Task created');
});

app.listen(3000, () => {
    console.log('Server running on <http://localhost:3000>');
});

2. Middleware

Middleware are functions that execute during the request-response cycle. They can perform operations such as logging, authentication, parsing request bodies, etc.

How It Works:

  • Use app.use() to apply middleware.
  • Middleware functions receive req, res, and next parameters.

Example:

const logger = (req, res, next) => {
    console.log(`${req.method} request to ${req.url}`);
    next(); // Call the next middleware or route handler
};

app.use(logger);

app.get('/tasks', (req, res) => {
    res.send('List of tasks');
});

Usage in the Task Management Application: Middleware can be used for logging, error handling, or parsing JSON bodies.

3. Error Handling

Error handling middleware can be defined to catch errors that occur in your application.

How It Works:

  • Define a middleware function with four parameters: err, req, res, next.
  • Use next(err) to pass errors to the error handling middleware.

Example:

app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).send('Something broke!');
});

Usage in the Task Management Application: You can use error handling to manage unexpected errors gracefully.

4. Template Engines

Template Engines allow you to render dynamic HTML pages. Express supports various template engines like EJS, Pug, and Handlebars.

How It Works:

  • Set the view engine using app.set('view engine', 'ejs').
  • Render templates with res.render().

Example:

app.set('view engine', 'ejs');

app.get('/tasks', (req, res) => {
    const tasks = ['Task 1', 'Task 2', 'Task 3'];
    res.render('tasks', { tasks });
});

Usage in the Task Management Application: You can use a template engine to render a list of tasks.

5. Static Files

Express can serve static files (like images, CSS, and JavaScript) using the built-in express.static() middleware.

How It Works:

  • Use app.use(express.static('public')) to serve static files from a directory.

Example:

app.use(express.static('public'));

Usage in the Task Management Application: You can place CSS files in the public folder and link them in your HTML templates.

6. Database Integration

Integrating a database allows you to persist data. Popular choices include MongoDB, MySQL, and PostgreSQL. For this example, we’ll use MongoDB with Mongoose as an ORM.

How It Works:

  • Install Mongoose: npm install mongoose.
  • Connect to the database and define models.

Example:

const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost:27017/taskdb', { useNewUrlParser: true, useUnifiedTopology: true });

const taskSchema = new mongoose.Schema({
    name: String,
    completed: Boolean
});

const Task = mongoose.model('Task', taskSchema);

// Create a new task
app.post('/tasks', async (req, res) => {
    const newTask = new Task({ name: 'New Task', completed: false });
    await newTask.save();
    res.send('Task created');
});

Project: Task Management Application

Step 1: Setup the Project

  1. Create a new directory for your project.
    mkdir task-management-app
cd task-management-app
npm init -y
npm install express ejs mongoose

Step 2: Create the Application

app.js: Main application logic

const express = require('express');
const mongoose = require('mongoose');
const app = express();

// Connect to MongoDB
mongoose.connect('mongodb://localhost:27017/taskdb', { useNewUrlParser: true, useUnifiedTopology: true });

// Task Schema
const taskSchema = new mongoose.Schema({
    name: String,
    completed: Boolean
});
const Task = mongoose.model('Task', taskSchema);

// Middleware
app.use(express.json());
app.use(express.urlencoded({ extended: true }));
app.use(express.static('public'));

// Set view engine
app.set('view engine', 'ejs');

// Logging Middleware
app.use((req, res, next) => {
    console.log(`${req.method} request to ${req.url}`);
    next();
});

// Routes
app.get('/tasks', async (req, res) => {
    const tasks = await Task.find();
    res.render('tasks', { tasks });
});

app.post('/tasks', async (req, res) => {
    const newTask = new Task({ name: req.body.name, completed: false });
    await newTask.save();
    res.redirect('/tasks');
});

// Error Handling
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).send('Something broke!');
});

// Start server
app.listen(3000, () => {
    console.log('Server running on <http://localhost:3000>');
});
