Notes on Deployment

Here’s a comprehensive guide on setting up a development workflow with Git version control, CI/CD pipelines, environment variables, server configuration, domain setup, and SSL certificates. Each topic is explained step-by-step, along with relevant code snippets and configurations.

1. Git Version Control

Git is a distributed version control system that helps track changes in source code during software development.

Installation

  1. Install Git:
    • On macOS: Use Homebrew
      brew install git
    • On Windows: Download from git-scm.com.
    • On Linux: Use your package manager
      sudo apt-get install git

Basic Commands

  1. Initialize a Repository:
    git init
  1. Clone an Existing Repository:
    git clone <repository-url>
  1. Add Files:
    git add <file-name> # or use git add . to add all files
  1. Commit Changes:
    git commit -m "Commit message"
  1. Push Changes to Remote:
    git push origin <branch-name>
  1. Pull Changes from Remote:
    git pull origin <branch-name>

Branching

Branching allows you to develop features independently from the main codebase.

  1. Create a New Branch:
    git checkout -b <branch-name>
  1. Switch to a Branch:
    git checkout <branch-name>
  1. Merge a Branch:
    git checkout main
git merge <branch-name>

2. CI/CD Pipelines

Continuous Integration (CI) and Continuous Deployment (CD) automate the process of testing and deploying applications.

Popular CI/CD Tools

  • GitHub Actions
  • Travis CI
  • CircleCI
  • Jenkins

Example: GitHub Actions Setup

  1. Create a GitHub Action Workflow:
    • Create a .github/workflows/ci.yml file in your repository.
name: CI/CD Pipeline

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v2

      - name: Set up Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '14'

      - name: Install dependencies
        run: npm install

      - name: Run tests
        run: npm test

      - name: Deploy to Production
        run: npm run deploy
        env:
          API_KEY: ${{ secrets.API_KEY }}

Explanation

  • Checkout code: Checks out the repository code.
  • Set up Node.js: Installs the specified Node.js version.
  • Install dependencies: Installs project dependencies.
  • Run tests: Executes tests.
  • Deploy to Production: Deploys the application, using environment variables stored in GitHub Secrets.

3. Environment Variables

Environment Variables are key-value pairs that provide configuration options and sensitive information without hardcoding them into your application.

Usage in Node.js

  1. Install dotenv:
    npm install dotenv
  1. Create a .env file:
    DATABASE_URL=mongodb://localhost:27017/mydatabase
PORT=3000
JWT_SECRET=mysecret
  1. Load Environment Variables in Your Code:
require('dotenv').config();

const dbUrl = process.env.DATABASE_URL;
const port = process.env.PORT;
const jwtSecret = process.env.JWT_SECRET;

Accessing Environment Variables

Use process.env.VARIABLE_NAME to access your variables within your application.

4. Server Configuration

Server Configuration involves setting up and managing your web server to handle requests and serve your application.

Example: Configuring an Express Server

  1. Basic Express Server Setup:
const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;

app.use(express.json());

app.get('/', (req, res) => {
    res.send('Hello, World!');
});

app.listen(PORT, () => {
    console.log(`Server is running on port ${PORT}`);
});
  1. Using Middleware:
    You can use middleware for logging, parsing, and authentication.
app.use((req, res, next) => {
    console.log(`${req.method} request for '${req.url}'`);
    next();
});

5. Domain Setup

Domain Setup involves registering a domain name and linking it to your web server.

Steps to Set Up a Domain

  1. Register a Domain:
    Use registrars like Namecheap, GoDaddy, or Google Domains.
  1. Point the Domain to Your Server:
    • Get your server’s IP address.
    • Update DNS records (usually an A record) at your domain registrar to point to your server’s IP.

Example DNS Record

Type: A
Host: @
Value: 192.0.2.1
TTL: 3600

6. SSL Certificates

SSL Certificates secure the data transmitted between a user's browser and your server.

Installing SSL Certificates

  1. Using Let’s Encrypt (Free SSL Certificates):
    • Install Certbot:
    sudo apt install certbot
    • Obtain a Certificate:
    sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
  1. Automatic Renewal:
    Certbot sets up a cron job to renew certificates automatically.

Verify SSL Installation

After installation, verify your SSL certificate using:

curl -I <https://yourdomain.com>

You should see HTTP/2 200 or similar status codes if everything is set up correctly.

Summary

This guide provides a foundational understanding of essential topics for setting up a web application. By mastering Git for version control, setting up CI/CD pipelines for automated testing and deployment, managing environment variables for secure configuration, configuring servers, setting up domains, and implementing SSL certificates, you’ll be well-equipped to handle modern web development workflows.
